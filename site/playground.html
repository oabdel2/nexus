Nexus computes a complexity score from 0.0 to 1.0 using six weighted signals:

        
score = keyword×0.30 + length×0.15 + role×0.15 + structure×0.15 + budget×0.10 + density×0.15

        
Keyword Score — matches the prompt against high-complexity keywords (analyze, architect, security…), medium keywords (compare, explain, summarize…) and low keywords (list, hello…). A density bonus rewards prompts with many keyword matches.

        
Length Score — longer prompts generally require more capable models. <20 chars → 0.05, <50 → 0.15, <200 → 0.3, <500 → 0.6, <1000 → 0.8, else 1.0.

        
Role Weight — different roles have different baseline complexity. Security analysts (0.9) and architects (0.85) skew higher; support (0.3) skews lower.

        
Structure Score — detects bullet points, code blocks, multi-line content, multiple clauses, and complex punctuation.

        
Budget Pressure — as budget depletes, pressure to use cheaper models rises: 1.0 − (remaining / 100).

        
Keyword Density — bonus for prompts matching many complexity-indicating keywords: min((highMatches + medMatches×0.5) / 3, 1.0).

        
The final score selects a tier: Economy (<0.21), Cheap (<0.35), Mid (<0.56), Premium (≥0.56).
