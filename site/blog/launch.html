Introducing Nexus: The AI Gateway That Learns Which Model to Use

July 2025 · 8 min read

The Problem: Every Request Gets the Expensive Model

Enterprise AI spending is projected to reach $407 billion by 2027, and a massive chunk of that spend goes directly to LLM API calls. Teams spin up ambitious agentic workflows — code generation, customer support, data analysis — and every single request gets routed to the most powerful (and most expensive) model available. A "Hello, world" message costs the same per-token as a complex debugging session. Multiply that across millions of daily requests and you have a cost structure that doesn't scale.

The pain is real: engineering teams report that 60–80% of their LLM requests are routine — simple lookups, formatting tasks, short Q&A — that could be handled perfectly well by a smaller, cheaper model. But nobody has time to manually classify every request and route it to the right tier. So the default behavior wins: everything goes to GPT-4o, the bill climbs, and finance starts asking hard questions.

Existing "AI gateways" don't solve this. They're proxies — they forward your request to the model you specify and maybe add some logging. They don't look at what you're asking. They don't know that your request is simple enough for a model that costs 10× less. They're plumbing, not optimization. We built Nexus to change that.

What Nexus Does

Change One Line, Save Up to 50%

Nexus is an OpenAI-compatible inference gateway. Point your existing SDK at Nexus instead of OpenAI directly — one line change — and Nexus handles the rest: classifying request complexity, selecting the optimal model tier, caching repeated queries, and compressing prompts before they leave your network.

Before (direct to OpenAI)
curl https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer sk-..." \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o","messages":[{"role":"user","content":"What is 2+2?"}]}'
After (through Nexus — one line change)
curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"auto","messages":[{"role":"user","content":"What is 2+2?"}]}'

# X-Nexus-Cost: 0.000015         ← actual cost tracked

The Request Pipeline

Every request flows through a 10-step optimization pipeline before it ever hits a model provider. Each step reduces cost or improves routing accuracy:

1Authenticate
2Validate
3Compress
4Cache Check
5Classify
6Route
7Cascade
8Send
9Evaluate
10Cache Store

Compress strips redundant whitespace and truncates overlong histories before tokens are billed. Cache Check queries a 3-layer cache — exact hash match (L1), BM25 keyword similarity (L2a), and semantic embedding similarity (L2b) — returning cached responses in under a millisecond. Classify scores request complexity using a TF-IDF classifier with 5 weighted signals. Route maps the complexity score to a model tier. Cascade optionally tries the cheap model first and auto-escalates if the confidence score is below threshold. Evaluate scores the response quality with a 6-signal heuristic so the system learns over time.

Adaptive Classifier

The routing engine uses a TF-IDF-based complexity classifier with 5 weighted signals: keyword analysis, context length, agent role, workflow step position, and budget pressure. Classification runs in ~250 ns for simple prompts and ~1.3 µs for complex ones — effectively zero overhead. The classifier supports online learning: a per-task-type confidence map adjusts routing thresholds over time based on observed response quality.

3-Layer Semantic Cache

Cache lookups cascade through three layers. L1 (exact) uses SHA-256 hashing — cache hits return in ~300 ns. L2a (BM25) catches paraphrased versions of the same question using keyword-based fuzzy matching. L2b (semantic) uses embedding similarity for true semantic deduplication. Enhancement features include a reranker verifier, synonym learning with persistence, and a shadow mode for safe cache rollout. Overall cache coverage is tested at 71.2%.

Security

The gateway runs a 17-middleware security chain: panic recovery, body size limits, request timeouts, security headers, request IDs, logging, CORS, IP allowlisting, rate limiting, OIDC SSO, input validation, prompt injection guard (16 regex patterns), RBAC, and audit logging. API keys are SHA-256 hashed. There's no telemetry phoning home, no data sharing, no third-party services. Your prompts never leave your infrastructure.

Engineering Discipline

The entire project compiles to a single static Go binary at 8.88 MB (stripped). There's exactly one external dependency: gopkg.in/yaml.v3 for config parsing. Everything else — the HTTP server, BM25 engine, TF-IDF classifier, circuit breaker, rate limiter, Prometheus exporter, distributed tracing — is built on the Go standard library. The codebase has 22,110 source lines and 18,506 test lines (a 1.19:1 source-to-test ratio), with zero TODO/FIXME/HACK comments and zero commented-out code. All 20 packages pass the race detector cleanly.

Try It Now

Get Nexus running in 60 seconds:

Docker (one line)
docker run -p 8080:8080 -e OPENAI_API_KEY=sk-... ghcr.io/oabdel2/nexus
From source
git clone https://github.com/oabdel2/nexus.git && cd nexus
go build -o nexus ./cmd/nexus/
./nexus serve

Then send a request:

curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"auto","messages":[{"role":"user","content":"Hello!"}]}'

Open http://localhost:8080/dashboard to see your savings in real time.
