Nexus Documentation

Everything you need to deploy, configure, and operate the Nexus intelligent AI inference gateway.

Getting Started

Go from zero to optimized AI inference in 3 steps.

Step 1 — Install

# Docker Compose (recommended)
git clone https://github.com/nexus-ai/nexus && cd nexus
docker compose up -d

# Or install the CLI
curl -fsSL https://get.nexus.ai | sh

Step 2 — Configure

Run the interactive wizard or create nexus.yaml manually:

# Interactive wizard
nexus init

# Or minimal config:
server:
  port: 8080
providers:
  - name: openai
    api_key: ${OPENAI_API_KEY}
    models: [gpt-4o, gpt-4o-mini]
  - name: anthropic
    api_key: ${ANTHROPIC_API_KEY}
    models: [claude-sonnet-4-20250514, claude-haiku]

Step 3 — Send Your First Request

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer nxs_live_abc123" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello world"}]
  }'
💡 Tip

Nexus is fully OpenAI-compatible. Just change your base_url to point at Nexus and all existing code works instantly.

Configuration Guide

Nexus is configured via nexus.yaml. Every section is optional with sensible defaults.

server

server:
  port: 8080              # Listen port (default: 8080)
  read_timeout: 30s       # Max request read time
  write_timeout: 120s     # Max response write time (includes LLM latency)
  max_concurrent: 0       # Max concurrent requests (0 = unlimited)

providers

providers:
  - name: openai
    type: openai             # openai, anthropic, or ollama
    base_url: https://api.openai.com/v1
    api_key: ${OPENAI_API_KEY}
    enabled: true
    priority: 1              # Lower = preferred
    models:
      - name: gpt-4o-mini
        tier: cheap           # economy, cheap, mid, or premium
        cost_per_1k_tokens: 0.00015
        max_tokens: 16384
      - name: gpt-4o
        tier: mid
        cost_per_1k_tokens: 0.005
        max_tokens: 16384
      - name: o3
        tier: premium
        cost_per_1k_tokens: 0.01
        max_tokens: 100000
  - name: anthropic
    type: anthropic
    base_url: https://api.anthropic.com/v1
    api_key: ${ANTHROPIC_API_KEY}
    enabled: false
    priority: 2
    models:
      - name: claude-sonnet-4
        tier: mid
        cost_per_1k_tokens: 0.003
        max_tokens: 64000

    type: ollama

Caching (3-Layer Semantic Cache)

Nexus uses a 3-layer cache with semantic learning to maximize hit rates while maintaining response quality. Three core cache layers (L1 exact, L2 BM25, L3 semantic) are enhanced by reranker verification, synonym learning, feedback loops, and shadow validation.

LayerTypeMatch MethodSpeedAccuracy
L1Exact MatchSHA-256 hash of normalized prompt< 1ms100%
L2aBM25 KeywordBM25 keyword similarity (threshold: 15.0)2–3ms~95%
L2bSemanticBGE-M3 embedding cosine similarity (threshold: 0.70)3–5ms~93%
RerankerCross-encoderBGE-reranker-v2-m3 verification for uncertain matches10–20ms~97%
SynonymQuery expansionAuto-discovered and manually added synonym mappings< 1ms~90%
FeedbackQuality loopQuality signals adjust cache confidence over time0msAdaptive
ShadowValidationCompare cache vs fresh responses for quality assuranceBackgroundAudit

Tuning Tips

  • Raise semantic threshold (0.70 → 0.85) if you see incorrect cache hits on similar but different prompts.
  • Lower BM25 threshold (15.0 → 12.0) for higher hit rates on keyword-heavy prompts (e.g., code questions).
  • Enable prefetch for sequential workflows where step N+1 is predictable from step N.
  • Set TTL per use case: short (5m) for rapidly changing data, long (24h) for stable knowledge questions.
💡 Cache Hit Identification

Check the X-Nexus-Cache response header. Values: l1_exact, l2_bm25, l2_semantic, l3_cluster, l4_stale, l5_prefetch, l6_compressed, or empty on miss.

Routing & Classification

Nexus routes every request to the optimal model tier using a TF-IDF based complexity classifier.

Complexity Scoring Formula

Each prompt is scored on a 0–1 scale using weighted features:

complexity = w1*token_count_norm + w2*vocab_richness
           + w3*code_density + w4*question_depth
           + w5*domain_specificity

Tier assignment:
  score < 0.3  =  fast     (gpt-4o-mini, claude-haiku, llama3)
  score < 0.7  =  mid      (claude-sonnet-4-20250514, gpt-4-turbo)
  score ≥ 0.7  =  premium  (gpt-4o, claude-opus)

TF-IDF Classifier

The classifier uses Term Frequency – Inverse Document Frequency to identify domain-specific vocabulary. Prompts containing rare technical terms (e.g., “monad transformer”, “CRDT”) score higher on domain specificity.

Cascade Behavior

When the eval system detects low confidence from a cheaper model, Nexus cascades to the next tier:

  1. Request starts at the classified tier (e.g., fast)
  2. Model returns a response with confidence score
  3. If confidence < cascade.confidence_threshold (default: 0.7), Nexus escalates to mid
  4. If still below threshold, escalates to premium
  5. Max escalations controlled by cascade.max_escalations
⚠️ Cascade adds latency

Each escalation adds a full round-trip. For latency-sensitive use cases, set max_escalations: 1 or start at a higher tier.

Troubleshooting

Common Errors

ErrorCauseFix
401 Unauthorized Missing or invalid API key Ensure Authorization: Bearer nxs_live_... header is set. Generate a key with POST /api/keys/generate.
429 Too Many Requests Rate limit exceeded Increase security.rate_limit.requests_per_minute or add burst capacity.
502 Bad Gateway All providers down (circuit breakers open) Check GET /api/circuit-breakers. Verify provider API keys. Wait for half-open recovery.
503 Service Unavailable Gateway not ready Check GET /health/ready. Ensure providers are configured and cache backend is accessible.
413 Payload Too Large Request exceeds max_body_size Increase server.max_body_size or compress prompts with the compression middleware.
Cache always misses Cache not configured or TTL expired Verify cache.enabled: true. Check TTLs. Verify cache backend (Redis/Qdrant) is connected.
Model not downrouting Classifier confidence too low Check /api/eval/stats for confidence scores. Lower router.smart_classifier.threshold_low.
Cascade loops All tiers returning low confidence Set cascade.max_escalations: 1. Review shadow eval stats for model quality issues.
High latency on first request Cold cache + TF-IDF model loading Normal on first start. Use /health/ready to wait for warmup. Consider predictive prefetch.
PII blocked unexpectedly Prompt guard false positive Lower security.prompt_guard.max_risk_score or add patterns to allowlist.
